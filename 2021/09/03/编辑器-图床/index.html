typora+picgo+github = 编辑器+图床

Typora：完成markdown文件的编辑
PicGo：上传图片至图床
Github：新建一个“仓库”作为图床

创作立场声明：MD 虽然有特定的语法，但其实很容易的。唯一麻烦的就是对图片的处理，此篇为架设免费图床的笔记。

简介：

  • Typora：Markdown 编辑器
  • PicGo：图床管理工具，一个用于快速上传图片并获取图片URL链接的工具
  • 支持图床：Gitee图床、Github图床、七牛云、阿里云OSS等
  • Github：代码托管仓库（本次作为图片仓库使用）

在Github建立图片仓库

Github链接：github.com

  • 进入GitHub，自行注册登录，然后点击右上角“+”
  • 点“新建仓库”
  • 自定义一个仓库名称
  • 设置为“公开”
  • 点创建

  • 创建好账号后
  • 右上角点击 “Setting”-“Developer settings”-“Personal access tokens”

  • 点击“Generate new token”

  1. 创建用户名。
  2. 选择有效期，根据自己选择，可以选择7天、30天、90天等，我选择没有有效期。
  3. 选择”repo”中所有的选项。
  4. 然后点击“Generate token”确认。

得到的 token 需自行保存，后面要用

安装 Markdown 编辑器：Typora

Typora链接：www.typora.io

安装及常规使用跳过，这里主要讲图床配置：

  • 左上角”文件 “，” 偏好设置 “，” 图像“
  • 选择” 上传图片 “，勾选：
  1. “对本地位置的图片应用上述规则”

上传服务 选”PicGo(app)“，点击” 下载 PicGo(app)“，并安装，然后将 PicGo 的安装路劲也填上

  • 在 PicGo 中，点” 图床设置 “，选择”github“图床。

  1. 仓库名：填写刚刚创建的仓库地址

  1. 设置分支名，一般默认为“main”，可以在仓库查找。

  1. 设置token：个人令牌，就是之前在 github 中生成私人令牌时得到的一串字符串，填入即可，“确定”。

  • 现在就可以去 Typora 中测试一下图床了
